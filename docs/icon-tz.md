# ТЗ: иконка для YT-sub

## 1. Контекст

YT-sub — macOS-утилита, живёт в menu bar. Достаёт метаданные и транскрипт YouTube-роликов и отдаёт их MCP-агентам. Распространяется как `.dmg` через GitHub Releases, ставится в `/Applications`.

Сейчас иконка генерируется кодом в `icon.py` (Cocoa-drawing): красный скруглённый прямоугольник с белым play-треугольником внутри, отдалённо похожий на логотип YouTube. Это плейсхолдер; нужна полноценная иконка от дизайнера.

## 2. Цель

Получить набор файлов, который заменит автогенерируемые ассеты в `assets/` и обеспечит правильное отображение во всех контекстах: menu bar, Finder, Dock, DMG, GitHub.

## 3. Где появляется иконка

| Контекст | Размер на экране | Источник |
|---|---|---|
| **Menu bar** (строка меню сверху) | 22×22 pt (44×44 на retina) | `assets/yt_icon.png` (PNG, цветной, **не template**) |
| **Finder /Applications, Dock, Spotlight** | до 1024×1024 px | `assets/yt_icon.icns` (multi-resolution iconset) |
| **DMG-окно при монтировании** | 128×128 px | тот же `.icns` |
| **About-диалог в трее** | 64×64 px | переиспользует `.png` |
| **GitHub README / превью репо** | 460×460 (для og:image), произвольный для inline | `docs/icon-1024.png` |

## 4. Размеры в `.icns`

Apple iconset spec (см. `iconutil`):

```
icon_16x16.png        16×16
icon_16x16@2x.png     32×32
icon_32x32.png        32×32
icon_32x32@2x.png     64×64
icon_128x128.png      128×128
icon_128x128@2x.png   256×256
icon_256x256.png      256×256
icon_256x256@2x.png   512×512
icon_512x512.png      512×512
icon_512x512@2x.png   1024×1024
```

Каждый PNG отрисовывается отдельно, не масштабируется автоматически — на 16×16 нужно убирать мелкие детали, упрощать форму. Поставлять все 10 файлов; сборка в `.icns` делается через `iconutil -c icns yt_icon.iconset`.

## 5. Размеры для menu bar

Отдельный PNG, не из iconset. macOS рендерит его в строке меню как 22×22 pt, поэтому нужно две версии:

- `yt_icon.png` — 22×22 px (1x)
- `yt_icon@2x.png` — 44×44 px (retina)

Менюбар-иконка должна оставаться читаемой при тёмной и светлой теме. Если иконка цветная (как сейчас) — она везде показывается одинаково; если template (чёрно-белая с альфа-маской) — macOS автоматически инвертирует под тему. Для YouTube-стиля цветная предпочтительна — узнаваемее.

## 6. Стиль

### 6.1 macOS Big Sur+ canvas guideline

Apple ввёл единый «squircle» canvas для всех app-иконок: размытие 13.3 % от стороны, padding ~10 % от каждой стороны, белый/прозрачный фон под формой. Дизайн **не должен заполнять всю площадь PNG** — оставлять воздух по краям. Без этого иконка визуально «бьётся» с системными в Dock.

См. [Apple Human Interface Guidelines → App icons](https://developer.apple.com/design/human-interface-guidelines/app-icons).

### 6.2 Визуальный язык

- Primary: что-то ассоциирующееся с YouTube + текстом/субтитрами (наша утилита достаёт **транскрипт**, не только видео). Идеи:
  - YouTube-style красный прямоугольник + белый play-треугольник + полоски текста под ним
  - Speech bubble с play-кнопкой
  - Лента субтитров
- Цвета:
  - Красный YouTube: `#FF0000` (или его близкий аналог, чтобы не нарушать TM — `#E60000`, `#D9001A`)
  - Белый акцент
  - Опционально: тёмно-серый для контраста на светлом фоне macOS
- Скруглённые углы. Без резких stroke-ов.
- Никаких теней внутри иконки на размерах ≤ 64×64 — съедают читаемость.

### 6.3 Что нельзя

- **Точная копия YouTube-логотипа** = нарушение их Trademark Guidelines. Стилизация ОК, литеральное копирование — нет.
- Текст внутри иконки на размерах < 128×128 — нечитаемо.
- Прозрачный фон без squircle-canvas — иконка будет «висеть в воздухе» в Dock.
- Цвета зависящие от тёмной темы (нет автоматического свитча для не-template иконок).

## 7. Формат файлов

Поставлять три набора:

1. **Исходник** — `docs/icon-source.{sketch|fig|svg|ai}`. Векторный, чтобы можно было перерендерить под любой размер.
2. **Menu bar** — `assets/yt_icon.png` (22×22) и `assets/yt_icon@2x.png` (44×44).
3. **App icon** — `assets/yt_icon.iconset/` со всеми 10 PNG по спецификации п. 4. Финальный `.icns` собирается локально через `iconutil` в `release.sh`.

PNG: 8-бит RGBA, без EXIF.

## 8. Замена автогенерируемых ассетов

Сейчас `icon.py` рендерит PNG в Cocoa и пишет в `assets/`. После получения дизайна:

1. Удалить функции `_render_png`, `ensure_icon` (rendering часть). Оставить `_assets_dir()` для лукапа.
2. PNG/iconset положить в репо (gitignore сейчас исключает `assets/yt_icon.iconset/` и `.icns` — снять исключение).
3. `setup.py DATA_FILES` уже включает `assets/yt_icon.png` и `assets/yt_icon.icns`, его трогать не нужно.

## 9. Приёмочные критерии

- [ ] В menu bar иконка чётко различима 22×22, не сливается с системными иконками macOS, читается на светлом и тёмном фоне строки меню.
- [ ] В Dock и Finder выглядит «родной» — соответствует Big Sur+ canvas, padding и скругление.
- [ ] На 16×16 иконка остаётся узнаваемой (не превращается в красное пятно).
- [ ] На 1024×1024 нет видимой пикселизации, формы остаются гладкими.
- [ ] DMG при монтировании показывает иконку в окне copy-to-Applications.
- [ ] `spctl --assess` проходит после подписи + нотаризации (формальное требование к bundle, не к дизайну, но проверить нужно).
- [ ] Не нарушаются YouTube Brand Guidelines (стилизация без копирования логотипа).
